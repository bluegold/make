# AGENT.md: タスクランナー自作課題 (Task Runner Challenge)

## 1. 概要

本プロジェクトの目的は、`Makefile` のサブセットを解析・実行する独自のタスクランナーを構築することです。この課題を通じて、**ファイルの依存関係（グラフ構造）の解決**、**構文解析**、および**再帰的なアルゴリズム**の実装方法を学びます。

## 2. コア要件

以下の 3 つの主要コンポーネントを実装してください。

### A. パーサ (Parser)

独自の定義ファイル（例：`Taskfile`）を読み込み、メモリ上のデータ構造に変換します。

- `ターゲット: 依存タスク1 依存タスク2 ...` の形式を解釈すること。
- ターゲットの次の行にある実行コマンド（インデントされた行）を関連付けること。

### B. 依存関係解決エンジン

ターゲットを実行する前に、その依存タスクを正しい順序で実行するロジックを実装します。

- **トポロジカルソート:** 依存グラフを解析し、実行順序を決定します。
- **循環参照検知:** `A -> B -> A` のような無限ループを検知し、エラーとして報告します。

### C. 実行エンジン

- 決定された順序に従って、シェルコマンドをサブプロセスとして実行します。
- **実行コマンドの表示:** 実行前に、実行するコマンドを `Executing: [command]` という形式で標準出力に表示してください。
- **タイムスタンプ比較 (発展):** ターゲットファイルが依存ファイルより新しい場合は実行をスキップするロジックを実装してください。

## 3. レベル別チャレンジ

### Level 1: 基本機能

- 変数のない単純な `Target: Dependency` 構造の解析。
- 深さ優先探索 (DFS) による順序解決と実行。

### Level 2: 変数展開 (Variable Expansion)

- `VAR = value` の形式で変数を定義できる。
- `$(VAR)` の形式でコマンド内で変数を利用できる。
- **重要:** 遅延評価（実行時に展開する）と即時評価の違いを意識して実装してください。

### Level 3: 特殊変数と高度な制御

- `$@` (現在のターゲット名) や `$<` (最初の依存ファイル名) などの自動変数のサポート。
- 並行実行（依存関係のないタスクをマルチスレッド/プロセスで同時に走らせる）の実装。

### Level 4: 暗黙ルールと make らしさの拡張

- Pattern Rule (`%.o: %.c` など) による暗黙ルール探索を実装すること。
- 明示ルールを最優先し、次に Pattern Rule を探索すること。
- 複数の候補がある場合は、より具体的なルールを優先すること。
- `.PHONY` をサポートし、ファイル存在やタイムスタンプに依存しないターゲットを扱えるようにすること。
- 実行対象が明示ルールで定義されていない場合でも、利用可能なルールから必要な依存関係を補完できるようにすること。
- 可能なら、ルール探索の結果を再帰的にたどれるようにして、単一段階だけでなく連鎖的な暗黙ルール解決を考慮すること。

## 4. 各言語での実装ヒント

実装言語の制限はありませんが、以下の言語特性を意識した設計を推奨します。

- **Rust / C++:** グラフ構造の所有権管理（Rc/Arc や Smart Pointer）の学習。
- **Go:** Goroutine と Channel を使った、依存関係に基づいた並行ビルドの実装。
- **Python / Ruby:** 動的なデータ構造（Dict/Hash）を活用した、簡潔なパーサの実装。
- **TypeScript (Node.js):** `child_process` と Promise を組み合わせた非同期コマンド制御。

## 6. 評価基準

1. 正確性: 依存順序が論理的に正しいか。
1. 堅牢性: 循環参照や存在しないタスクに対して、適切なエラーメッセージを出せるか。
1. 変数展開の整合性: 再帰的な展開（A=$(B), B=val）が正しく解決されるか。
1. 設計: パーサ、グラフ解決、エグゼキュータが適切にモジュール化されているか。

## 7. CLI インターフェース

プログラムは以下の形式で引数を受け取れるようにしてください。

```bash
$ [command] [taskfilename] [target]
```

- taskfilename (任意): 読み込む定義ファイル。省略された場合はカレントディレクトリの Taskfile を使用する。
- target (任意): 実行するターゲット名。省略された場合は、定義ファイル内で一番最初に記述されているターゲットを実行する。
- エラー処理: 指定されたファイルが存在しない場合や、ターゲットが見つからない場合は、適切なエラーメッセージを表示して終了すること。
