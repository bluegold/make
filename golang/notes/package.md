# package 分割の方針

Go 版のタスクランナーは、`package main` に処理を集めすぎない構成が望ましい。
実装の中心は共有パッケージに寄せ、各レベルの `main.go` は起動処理のみを担当する。

## 基本方針

- `golang/taskrunner` に共通処理を置く
- `level1`, `level2`, `level3` はそれぞれ薄い入口にする
- 解析、正規化、依存解決、実行を同一構造体に閉じ込めない

## 想定する責務

- `ParseFile`: Taskfile を読み、変数とタスクを構築する
- `NormalizeProgram`: 変数展開済みの実行向けデータに整える
- `ResolveDependencies`: 実行順を決める
- `ExecuteSequential` / `ExecuteParallel`: 実行方式を分ける

## 期待する効果

- level2 以降で共通処理を再利用しやすくなる
- テスト対象を関数単位に分けやすくなる
- `main.go` が小さくなり、レベル差分が見えやすくなる

## 実装上の注意

- `main` 側で `os.Exit` を呼ぶのは最終段のみとする
- データの更新は共有構造体の直接書き換えより、新しい値を返す形を優先する
- 入口レイヤーにレベル固有の分岐をため込まない
