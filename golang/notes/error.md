# error の扱い

Go 版では、エラーを途中で握りつぶさず、`return error` で上位へ伝える方針を採る。
終了処理は `main.go` に集中させる。

## 基本方針

- 解析エラーは `ParseFile` が返す
- 循環依存や未知ターゲットは `ResolveDependencies` が返す
- コマンド失敗は `Execute*` が返す
- `main.go` は受け取った `error` を表示して終了する

## エラー種別

- ファイル未存在
- 入力形式不正
- 未知ターゲット
- 循環依存
- 変数循環
- コマンド実行失敗

## 実装方針

- エラー文は、どの段階で失敗したか分かるようにする
- 必要な箇所では `fmt.Errorf("...: %w", err)` で文脈を足す
- `panic` は通常の制御フローに使わない
- `os.Exit` は `main` のみで使う

## 期待する効果

- テストしやすくなる
- 失敗点を切り分けやすくなる
- level2 / level3 でも同じエラー処理の型を使える
