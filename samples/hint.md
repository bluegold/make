# 実装のアドバイス (samples/hint.md)

このドキュメントでは、Makefile クローンを自作する際の手助けとなる技術的なヒントをレベル別にまとめています。

---

## Level 1: 依存関係の解決 (グラフ理論)

### 1. データ構造の設計

ターゲットと依存関係は、**「有向グラフ (Directed Graph)」**として保持すると扱いやすくなります。

- `Map<String, Task>` のような形で、ターゲット名をキーに、コマンドと依存リストを値に持つ構造を目指しましょう。

### 2. 実行順序の決定 (トポロジカルソート)

依存関係を正しく解決するには、以下の手順が有効です。

1. **深さ優先探索 (DFS):** 実行したいターゲットから再帰的に依存先を辿ります。
2. **ポストオーダー (帰りがけ順):** 全ての依存先を訪れた「後」で、そのターゲットを実行リストに追加します。
   - 例: `A: B` なら、まず `B` を解決し、その後に `A` をリストに入れます。

### 3. 循環参照の検知

探索中に「現在訪問中のタスク」を記録するスタックを持ち、探索が終わっていないタスクに再び出会ったら、それは循環（ループ）しています。

---

## Level 2: 変数展開 (文字列処理)

### 1. 変数の保存

ファイル読み込み時に `VAR = VALUE` の形式を見つけたら、専用の辞書（Environment Table）に保存します。このとき、値はまだ展開せず、文字列のまま保持するのが「遅延評価」の第一歩です。

### 2. 再帰的な展開のロジック

`$(VAR)` を見つけた際、以下の擬似コードのような再帰処理を行うと堅牢です。

```python
def expand(text, env):
    while "$( " in text:  # $ ( が含まれている間繰り返す
        var_name = find_inner_variable(text)
        value = env.get(var_name, "")
        text = text.replace("$(" + var_name + ")", value)
    return text
```

### 3. 特殊変数の注入

$@ や $< は、タスクを実行する直前に、そのタスクのコンテキスト（ターゲット名や依存リスト）に基づいて一時的に変数テーブルへ書き込むことで実装できます。

## Level 3: 最適化と高度な機能

### 1. タイムスタンプの比較による実行スキップ

ビルドツールとしての賢さは、不要な計算をしないことにあります。

- mtime (Modified Time) の取得
  OSのシステムコールを使用して、ターゲットファイルと依存ファイルの最終更新日時を取得します。

- 判定ロジック

1. ターゲットファイルが存在しない場合は、必ず実行します。
2. ターゲットより新しい依存ファイルが一つでもある場合は、実行します。
3. それ以外の場合は up to date として実行をスキップします。

### 2. シェルコマンドの実行とエラー制御

各言語のサブプロセス実行ライブラリを使用して、コマンドをシェル経由で実行します。

- 逐次停止
  依存タスクの一つが失敗（終了ステータスが 0 以外）を返した場合、後続のタスクや親タスクの実行を即座に中断するロジックが必要です。

- サイレントモード
  コマンドの先頭に @ がある場合、実行するコマンド文字列自体を標準出力に表示しないように制御します。

### 3. 自動変数の注入

$@（ターゲット名）や $^（全依存ファイル）などの自動変数は、タスクが実行キューに入る直前に、そのタスクのコンテキストから生成します。
実行エンジンがターゲットを処理する際に、一時的な変数テーブルにこれらの値をセットし、展開エンジンに渡します。

### 4. 並行実行 (Parallel Execution) の考え方

依存関係のないタスクを同時に実行することで、パフォーマンスを向上させます。

- 依存カウントの利用
  各タスクに 未完了の依存数（入次数）を持たせます。

- キュー管理
  依存数が 0 になったタスクを順次スレッドプールやワーカーに投入します。一つのタスクが終わるごとに、それを待ち受けていた後続タスクの依存カウントを減らしていきます。

## デバッグとテストのコツ

- ドライランモードの実装
  実際にコマンドを実行せず、実行されるはずのコマンドを順番に表示するだけの機能（make -n 相当）を作ると、ロジックのデバッグが楽になります。

- ログの可視化
  解析した依存グラフや、変数展開後の最終的なコマンド文字列をデバッグログとして出力できるようにしておきましょう。
