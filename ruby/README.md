# Ruby Task Runner Implementation

このディレクトリには、Ruby 4.0 で実装するタスクランナーの設計と実装方針を置きます。

このプロジェクトの Ruby 版は、他言語の写しではなく、Ruby らしい書き味を優先します。

- `Data` で小さく不変な値を表す
- `Set` で循環参照や訪問済み管理をする
- `case ... in` のパターンマッチングでパーサを組む
- レベル 3 では `Ractor::Port` を中心に並列実行を設計する

## 目標

- Makefile のサブセットを読み込んで実行する
- 依存関係を正しく解決する
- 変数展開を遅延評価で扱う
- 自動変数、タイムスタンプ判定、並列実行をサポートする

## レベル構成

- `level1`: ターゲットと依存関係の解決
- `level2`: `$(VAR)` の変数展開と循環参照検知
- `level3`: `$@` / `$<` / `$^`、タイムスタンプスキップ、並列実行

各レベルは独立した成果物として扱い、`ruby/level1/lib`、`ruby/level2/lib`、`ruby/level3/lib` のように分ける想定です。

テストも同様にレベルごとに配置し、実装と近い場所に `test` を置く方針にします。

level2 と level3 は、level1 で作ったクラスや関数を `require` して再利用してよいものとします。
共通処理は level1 に寄せ、上位レベルは変数展開や並列実行などの差分を足す構成にします。
